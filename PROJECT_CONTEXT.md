# Контекст Проекту: Календарний План (УЦХВЄ м. Івано-Франківська)

## 1. Загальний опис
Додаток створюється для планування розкладу церковних подій. 
Стек технологій: **React**, **Vite**, **Tailwind CSS**, та **Firebase** (в якості бази даних — Firestore).

## 2. Основний функціонал
- **Календар**: перегляд та редагування подій по днях, місяцях та роках.
- **Шаблони днів тижня**: система генерації типових подій для кожного дня тижня. Шаблони дозволяють налаштувати "стандартний" тиждень, з якого генеруються розклади на наступні місяці (завдяки `generateEventsFromTemplate`).
- **Списки (довідники)**: управління списками служителів (проповідників, регентів, гуртів), місць проведення, відділів тощо.
- **Telegram інтеграція**: можливість публікувати розклад у Telegram за допомогою бота (з використанням Bot Token та Chat ID).
- **Рольова модель (Доступи)**: 
  В додатку налаштований розподіл доступу через паролі для різних відділів:
  - **Адмін (admin)**: повний доступ до всіх меню, налаштувань, шаблонів.
  - **Пастор / ПРИЗНАЧЕННЯ ПРОПОВІДНИКІВ (preacher_manager)**: доступ лише до призначення служителів слова.
  - **Левит / ПРИЗНАЧЕННЯ ЛЕВИТІВ (singer_manager)**: доступ лише до музичної частини.

## 3. Останні відкриті питання та зміни
1. **Шаблони тижневих подій**: Обговорювався алгоритм збереження кастомних подій (створених вручну) при оновленні шаблону. Система визначає шаблонні події за полем `sourceTemplateEventId`.
2. **Баг з паролями доступу**: Було виявлено проблему, через яку старий пароль відкривав доступ до всіх ролей одночасно, а нові паролі не застосовувалися коректно до відповідних рівнів (`admin`, `preacher_manager`, `singer_manager`). Це завдання потребує допрацювання при відновленні роботи.

## 4. Подальші кроки (для нового чату)
Для продовження роботи асистент повинен звернути увагу на файл `src/App.tsx`, де знаходиться:
- Основна логіка відображення та авторизації (`handleAuthSuccess`, `passwordPrompt`).
- Логіка обробки шаблонів (`generateEventsFromTemplate`, `saveTemplate`).
- Редагування подій.
